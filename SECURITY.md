# Security Policy

## Política de Segurança e Divulgação de Vulnerabilidades

A segurança dos dados e a experiência dos usuários do OtimizeTour são prioridades. Se você encontrar uma vulnerabilidade em nosso sistema, pedimos que a reporte de forma responsável.

## Como Reportar uma Vulnerabilidade

Para garantir que a vulnerabilidade seja resolvida antes de ser explorada, pedimos que você nos notifique **diretamente e de forma privada**.

**Não use os issues públicos do repositório para relatar falhas de segurança.**

**Canal de Contato**
Envie seu relatório de vulnerabilidade para o e-mail:

- guilherme_h_nunes@estudante.sesisenai.org.br 
- miguel_carvalho@estudante.sc.senai.br 

**Informações Essenciais**

Por favor, inclua o máximo de detalhes possível no seu relatório:

1. **Descrição:** Qual é a vulnerabilidade (Ex: XSS, SQL Injection).

2. **Localização:** Qual página ou funcionalidade é afetada (Ex: Login, Geração de Roteiro).

3. **Passos para Reproduzir:** Instruções claras sobre como replicar a falha.

4. **Impacto:** O que pode ser feito por um atacante ao explorar esta falha.

## Nosso Compromisso

1. **Confirmação:** Iremos confirmar o recebimento do seu relatório em até 48 horas úteis.

2. **Ação:** A equipe trabalhará para avaliar, priorizar e corrigir a vulnerabilidade.

3. **Notificação:** Manteremos você informado sobre o progresso e notificaremos quando a correção for implementada.
